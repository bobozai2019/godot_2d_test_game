# 2D 阴阳师风格游戏计划

## 目标

制作一个小型 2D 俯视角动作 RPG 原型，主题参考阴阳师方向：玩家角色在紧凑战斗场景中探索，击败妖怪风格敌人，释放简单的符咒或法术能力，并通过最小化 RPG 界面查看状态和成长信息。

为了避免依赖其他游戏的 IP，这里的“阴阳师”只作为主题方向：日式灵异幻想、式神、符咒、灵体、神社或庭院环境，以及仪式法术。

## 首个可玩范围

1. 创建一个可玩的俯视角战斗场景。
2. 使用当前剑士素材包作为占位，添加一个可控制主角。
3. 使用现有史莱姆、植物、兽人素材包，添加三类敌人。
4. 使用动画魔法书素材包作为魔法物件或特效占位，添加一个法术或召唤能力。
5. 添加最小 HUD：生命值、技能冷却、敌人波次数、胜负界面。
6. 在主题地图素材补齐前，使用占位 tiles 或简单 Godot 图形搭建一个竞技场地图。

- `craftpix-net-284465-free-predator-plant-mobs-pixel-art-pack`：378 个 PNG、144 个 Aseprite、36 个 PSD。适合作为敌人族群，包含 idle、walk/run、attack、hurt 和 death 动画。
- `craftpix-net-363992-free-top-down-orc-game-character-pixel-art`：672 个 PNG、192 个 Aseprite、48 个 PSD。适合作为敌人族群，或精英怪/Boss 占位。

## 资源缺口

要让项目真正像阴阳师风格游戏，还需要补齐：

- 主题主角：阴阳师长袍、扇子、符咒、纸符或仪式法杖动画。
- 式神/妖怪伙伴：至少一个可召唤灵体，包含 idle、move、attack、hurt、death/vanish 动画。
- 日式灵异敌人：鬼、狐灵、纸人、幽灵、天狗、灯笼妖或面具灵。当前史莱姆/植物/兽人在机制上可用，但主题辨识度不强。
- 环境 tileset：神社庭院、鸟居、石板路、榻榻米房间、夜间森林、灯笼、水面、墙体、摆件，以及可配置碰撞的 tiles。
- 法术 VFX：符咒弹、召唤阵、阴阳爆发、封印命中特效、治疗/结界效果、敌人受击特效。
- 音频：脚步、命中、施法、UI 点击、敌人死亡、环境循环、战斗音乐。
- UI 主题化：当前 RPG UI 可用于布局，但还需要阴阳师风格边框、纸张质感、符咒图标和更贴合技能的图标。
- 字体：可读性好的中文/日式风格字体，并且许可证适合本项目使用。
- 立绘/对话资源：后续可选；如果游戏包含剧情、类抽卡角色界面或视觉小说式互动，就需要补齐。

## 开发阶段

### 阶段 1：项目基础

- 创建 `scenes/`、`scripts/`、`assets/` 和 `ui/` 文件夹。
- 创建 `main.tscn`、`player.tscn`、`enemy.tscn`、`hud.tscn` 和一个小型测试竞技场。
- 尽量通过 Godot 编辑器在 `project.godot` 中配置主场景。
- 添加输入动作：上/下/左/右移动、攻击、skill_1、skill_2、交互、暂停。

### 阶段 2：战斗原型

- 使用 `CharacterBody2D` 实现俯视角玩家移动。
- 接入剑士 idle/move/attack/hurt/death 占位动画。
- 实现敌人 AI：待机、追击、攻击、受伤、死亡。
- 添加敌人生成器和一个清波胜利条件。
- 添加简单的生命值/伤害/冷却数据资源。

### 阶段 3：阴阳师机制

- 添加符咒飞弹或短距离封印攻击。
- 使用魔法书素材添加一个召唤/法术占位能力。
- 添加敌人状态效果接口：减速、束缚、燃烧或破盾。
- 添加一个简单的阴阳能量槽：战斗中积累能量，并消耗能量释放技能。

### 阶段 4：UI 与循环

- 使用 RPG UI 包搭建 HUD。
- 添加暂停菜单和胜负面板。
- 添加结算界面，展示击败敌人数量和存活时间。
- 只在核心循环可玩之后再添加标题/菜单场景。

### 阶段 5：主题化

- 等主题素材补齐后，将占位剑士/兽人/史莱姆替换为阴阳师、式神和妖怪素材。
- 将占位竞技场替换为神社/庭院 tiles。
- 添加法术 VFX 和音频。
- 将素材命名整理到 `assets/characters/`、`assets/enemies/`、`assets/ui/`、`assets/vfx/` 和 `assets/audio/`。

## 推荐下一步

先用现有剑士、史莱姆、植物、兽人、魔法书和 RPG UI 素材启动阶段 1 与阶段 2。同时并行寻找或制作缺失的主题资源：阴阳师主角、神社 tileset、式神/妖怪、符咒 VFX 和音频。

编码前，如果 `__MACOSX`、`.DS_Store` 和 `._*` 这类 macOS 压缩包生成文件不是刻意保留的资源，应清理或忽略。
