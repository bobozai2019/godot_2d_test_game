# 2D 阴阳师 demo 游戏计划

## 目标

制作一个小型 2D 俯视角动作 RPG demo：玩家角色在紧凑战斗场景中移动、攻击、释放一个简单技能，击败几波敌人，并通过最小 HUD 查看生命值、技能冷却和胜负状态。

这里的“阴阳师”只作为玩法代号或灵感方向，不要求做成日式和风，也不追求完整主题包装。demo 优先验证移动、战斗、敌人、技能、UI 和胜负循环；所有现有资源能重复使用就重复使用。

## 首个可玩范围

1. 创建一个可玩的俯视角战斗场景。
2. 使用当前剑士素材包添加一个可控制主角；需要时可重复使用同一套动作资源。
3. 使用现有史莱姆、植物、兽人素材包添加敌人；demo 中可以复用同一敌人素材，通过血量、速度、颜色调制或缩放区分类型。
4. 使用动画魔法书素材包或现有角色攻击帧作为技能占位，添加一个法术、召唤或范围攻击能力。
5. 添加最小 HUD：生命值、技能冷却、敌人波次数、胜负界面。
6. 使用占位 tiles、现有 UI/tile 图或简单 Godot 图形搭建一个竞技场地图，不要求主题地图。

## 现有资源

`resources/` 下可用素材包：

- `craftpix-net-180537-free-swordsman-1-3-level-pixel-top-down-sprite-character`：728 个 PNG、192 个 Aseprite、48 个 PSD。适合作为占位主角，包含 idle、walk、run、attack、hurt、death 和阴影变体。
- `craftpix-net-255216-free-basic-pixel-art-ui-for-rpg`：40 个 PNG、26 个 PSD。可用于主菜单、按钮、行动面板、角色面板、背包、设置、商店、数字、胜负 UI、图标和 RPG 面板。
- `craftpix-net-284465-free-predator-plant-mobs-pixel-art-pack`：378 个 PNG、144 个 Aseprite、36 个 PSD。适合作为敌人族群，包含 idle、walk/run、attack、hurt 和 death 动画。
- `craftpix-net-363992-free-top-down-orc-game-character-pixel-art`：672 个 PNG、192 个 Aseprite、48 个 PSD。适合作为敌人族群，或精英怪/Boss 占位。
- `craftpix-net-788364-free-slime-mobs-pixel-art-top-down-sprite-pack`：318 个 PNG、144 个 Aseprite、36 个 PSD。适合作为前期敌人族群。
- `craftpix-net-809047-free-animated-magic-book-pixel-art-asset-pack`：152 个 PNG、18 个 Aseprite、24 个 PSD。可作为法术书、召唤媒介、掉落物或魔法 UI 图标占位。

这些资源已经足够支撑 demo，不需要为了 demo 阶段额外购买或制作专属主角、专属敌人、日式地图、完整 VFX 或音乐。

## 资源缺口

demo 阶段的资源缺口很小，优先按“能跑、能玩、能看懂”处理：

- 主角：现有剑士素材足够作为 demo 主角，不需要新做阴阳师外观。
- 敌人：现有史莱姆、植物、兽人素材足够；可以重复使用同一敌人，通过参数差异做出普通怪、快怪、厚血怪或精英怪。
- 技能表现：现有魔法书、角色攻击帧、简单 Sprite2D/ColorRect/粒子或 Godot 形状足够做占位。可暂时不做专属符咒、召唤阵、阴阳爆发等 VFX。
- 地图：可以用简单地面色块、碰撞墙、现有 UI/tile 图或 Godot 基础图形搭建 arena，不需要神社、鸟居、庭院等和风 tileset。
- UI：现有 RPG UI 包足够做 HP、冷却、波次和胜负面板，不需要主题化 UI。
- 音频：demo 可以先没有音乐，也可以完全没有音效；后续需要手感时再补 UI 点击、攻击、命中和死亡音效。
- 字体：先使用 Godot 默认字体或系统可用字体即可；只有出现中文显示不清晰时再补充可商用字体。
- 立绘/对话：demo 阶段不需要。

暂不列为 demo 必需缺口：

- 日式和风主角、式神、妖怪、神社地图、专属符咒 VFX、剧情立绘、完整音乐与环境音。

## 开发阶段

### 阶段 1：项目基础

- 创建 `scenes/`、`scripts/`、`assets/` 和 `ui/` 文件夹。
- 创建 `main.tscn`、`player.tscn`、`enemy.tscn`、`hud.tscn` 和一个小型测试竞技场。
- 尽量通过 Godot 编辑器在 `project.godot` 中配置主场景。
- 添加输入动作：上/下/左/右移动、攻击、skill_1、skill_2、交互、暂停。

### 阶段 2：战斗原型

- 使用 `CharacterBody2D` 实现俯视角玩家移动。
- 接入剑士 idle/move/attack/hurt/death 占位动画。
- 实现敌人 AI：待机、追击、攻击、受伤、死亡。
- 添加敌人生成器和一个清波胜利条件。
- 添加简单的生命值/伤害/冷却数据资源。

### 阶段 3：demo 技能机制

- 添加一个简单远程飞弹、短距离范围攻击或召唤占位能力。
- 优先使用魔法书素材、现有攻击帧或 Godot 基础形状做技能表现。
- 可选添加一个简单状态效果，例如减速或击退。
- 如实现成本低，再添加一个能量槽；否则 demo 阶段只保留技能冷却。

### 阶段 4：UI 与循环

- 使用 RPG UI 包搭建 HUD。
- 添加暂停菜单和胜负面板。
- 添加结算界面，展示击败敌人数量和存活时间。
- 只在核心循环可玩之后再添加标题/菜单场景。

### 阶段 5：可选打磨

- 根据试玩反馈决定是否替换主角、敌人或地图素材。
- 如果仍想保留阴阳师方向，再逐步补主角、式神、妖怪、符咒 VFX 或和风场景。
- 音频不是 demo 必需项，只在需要提升反馈手感时添加。
- 将素材命名整理到 `assets/characters/`、`assets/enemies/`、`assets/ui/`、`assets/vfx/` 和 `assets/audio/`。

## 推荐下一步

先用现有剑士、史莱姆、植物、兽人、魔法书和 RPG UI 素材启动阶段 1 与阶段 2。不要先找新资源，也不要先做主题化；等 demo 的移动、攻击、敌人、技能、HUD 和胜负循环跑通后，再判断是否需要新增素材。

编码前，如果 `__MACOSX`、`.DS_Store` 和 `._*` 这类 macOS 压缩包生成文件不是刻意保留的资源，应清理或忽略。
