# Project Memory

This file records project-specific operating conventions. Load and follow it at the start of each session for this repository.

## Required Workflow

1. Before starting any task, check the local Git status.
2. If there are uncommitted local changes before the task starts, commit them first.
3. Plan every task before implementation.
4. Write the plan result under `doc/` before executing the task.
5. Write all task plans in Chinese and save them locally under `doc/`.
6. Keep task plans concise and named clearly, for example `doc/2026-04-11-项目约定计划.md`.
7. After creating a Git commit, push the commit to the GitHub remote repository.

## Notes

- The project memory file must stay in the repository root.
- Do not put credentials, passwords, tokens, or machine-specific secrets in memory files, plans, commits, or remote URLs.
